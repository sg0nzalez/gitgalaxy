Welcome to the **COBOL to Java Translation Suite**. 

Most legacy modernization efforts fail because they feed raw, monolithic COBOL directly into an LLM and hope for the best. This guarantees hallucinations, memory leaks, and broken dependencies. 

GitGalaxy flips the paradigm. We use deterministic mathematical parsing to build a **100% compiling Java Spring Boot architecture first**. We map the exact memory boundaries, generate the PostgreSQL entities, wire the REST endpoints, and scaffold the build system. Once the shell is structurally perfect, we package the remaining complex business logic into strict JSON tickets for autonomous AI agents to complete. 

**🔥 Proven in the Field:** This pipeline has been stress-tested across 27 distinct legacy COBOL repositories using our automated `batch_test_harness.py`. The result? 27 from-scratch, structurally sound, Maven-compilable Spring Boot systems generated entirely without human intervention.

### 🧠 How It Works: Deterministic Scaffolding

The `cobol_to_java_controller.py` ingests the JSON Intermediate Representation (IR) generated by the COBOL Refractor. It then orchestrates a suite of specialized forges:

#### 1. Entity & Memory Mapping Forge
Translates legacy schema boundaries into strict Spring Boot `@Entity` classes.
* **Precision Mapping:** Translates complex `PIC S9(7)V99` clauses into exact `BigDecimal` scales.
* **Array Resolution:** Detects `OCCURS` clauses and generates `@ElementCollection` Lists.
* **Memory Overlays:** Detects legacy `REDEFINES` (where multiple variables share the same memory block) and safely maps them as `@Transient` aliases in Java.

#### 2. The API Contract & Service Forge
Translates the DAG lineage intent into modern REST Controllers and auto-wires the `@Service` layer.
* **Paradigm Detection:** If a module receives structured data (CICS), it builds a Transactional `@PostMapping` with DTOs. If it relies on physical files (JCL), it builds a Batch endpoint using `MultipartFile` streams.
* **Mock Service Shield:** To prevent missing external COBOL subroutines from crashing the Spring Application Context upon booting, the forge dynamically generates Mock Services to satisfy Dependency Injection until those modules are migrated.

#### 3. EBCDIC Decoder Forge
Automatically generates the utility classes necessary to read raw mainframe byte streams. It includes strict hex-boundary validation to safely unpack legacy `COMP-3` (Packed Decimal) data and EBCDIC strings without crashing on corrupt bytes.

#### 4. The Build System Forge
Generates a production-ready `pom.xml` (Java 17, Spring Boot 3.2.4, PostgreSQL) and `application.yml` so the generated microservice is instantly compilable via `mvn clean compile`.

#### 5. The AI Boundary (Agent Task Forge)
Once the compiling shell is built, the pipeline extracts the isolated business rules (sliced by the Microservice Slicer) and packages them into **Autonomous Agent Job Tickets**. 
* These strict JSON tickets provide the LLM with the target variables, the architectural warnings, and the required data inputs/outputs.
* It binds the AI to a strict system prompt: *Implement the logic, use the pre-wired interfaces, and do not hallucinate external systems.*
